# Getting Started with MCP-B

Welcome! If you've read about MCP-B's architecture, you're probably eager to start building. This guide will walk you through setting up your first MCP-B powered website.

## What You'll Build

By the end of this tutorial, you'll have:

- A web page that exposes MCP tools through `window.mcp`
- A working todo management system accessible via MCP
- A chat interface that can interact with your tools
- Understanding of how to add your own custom tools

## Prerequisites

Before starting, make sure you have:

- **Node.js 18+** installed on your machine
- **pnpm** or **npm** package manager
- Basic familiarity with **React** and **TypeScript**
- A text editor (VS Code recommended)

## Step 1: Project Setup

First, let's create a new project. MCP-B works with any modern web framework, but we'll use **Next.js** for this example:

```bash
# Create a new Next.js project
npx create-next-app@latest my-mcp-app

# Navigate to your project
cd my-mcp-app

# Install MCP-B dependencies
npm install @modelcontextprotocol/sdk @mcp-b/transports zod
```

## Step 2: Define Your MCP Tools

Create a file for your MCP server definition. This is where you'll define what tools your website exposes:

```typescript
// src/services/mcp.ts
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { TabServerTransport } from '@mcp-b/transports';
import { z } from 'zod';

// Create the MCP server
export const mcpServer = new McpServer({
  name: 'my-app-mcp',
  version: '1.0.0',
});

// Define your first tool
mcpServer.tool('get_greeting', 'Get a friendly greeting', {
  name: z.string().describe('Name to greet')
}, async ({ name }) => ({
  content: [{
    type: 'text',
    text: `Hello, ${name}! Welcome to MCP-B.`
  }]
}));

// Connect the server with Tab Transport
await mcpServer.connect(new TabServerTransport({
  allowedOrigins: ['*']
}));
```

## Step 3: Expose MCP to the Browser

Now let's make your tools available in your application:

```typescript
// app/layout.tsx or your entry point
'use client';

import { useEffect } from 'react';
import { mcpServer } from './services/mcp';

export default function RootLayout({ children }) {
  useEffect(() => {
    // Server will automatically connect via Tab Transport
    console.log('MCP Server initialized');
  }, []);

  return (
    <html>
      <body>{children}</body>
    </html>
  );
}
```

## Step 4: Test Your Setup

Start your development server:

```bash
npm run dev
```

Open your browser and:

1. Install the MCP-B extension from the Chrome Web Store
2. Visit your application (http://localhost:3000)
3. Click the MCP-B extension icon
4. You should see your `get_greeting` tool available!

## Common Patterns

### Adding More Tools

As your app grows, you'll want to add more tools. Here's a pattern for organizing them:

```typescript
// Add a todo tool
mcpServer.tool('create_todo', 'Create a new todo', {
  text: z.string().describe('Todo text'),
  priority: z.enum(['low', 'medium', 'high']).optional()
}, async ({ text, priority }) => {
  // Your todo creation logic here
  return {
    content: [{
      type: 'text',
      text: `Created todo: ${text} with priority ${priority || 'medium'}`
    }]
  };
});
```

### Error Handling

Always wrap your tool implementations with proper error handling:

```typescript
mcpServer.tool('risky_operation', 'A tool that might fail', {
  // schema
}, async (params) => {
  try {
    // Your tool logic here
    return {
      content: [{ type: 'text', text: 'Success!' }]
    };
  } catch (error) {
    return {
      content: [{
        type: 'text',
        text: `Error: ${error instanceof Error ? error.message : 'Unknown error'}`
      }],
      isError: true
    };
  }
});
```

## Dynamic Tool Registration

MCP-B supports dynamic tool updates. Tools can be added or removed based on application state:

```typescript
// Component that registers tools when mounted
function ShoppingCartTools({ cartId }) {
  useEffect(() => {
    const tool = mcpServer.tool('checkout', 'Checkout the cart', {
      paymentMethod: z.enum(['credit', 'debit', 'paypal'])
    }, async ({ paymentMethod }) => {
      // Process checkout
      return {
        content: [{ type: 'text', text: 'Order created!' }]
      };
    });

    // Cleanup when component unmounts
    return () => {
      // Tool is automatically removed
    };
  }, [cartId]);

  return null;
}
```

## Next Steps

## Troubleshooting

### Tools not showing in extension

Verify that:
1. The MCP-B extension is installed and enabled
2. Your Tab Transport is properly initialized
3. Check the browser console for any errors
4. Ensure `allowedOrigins` includes your domain

### TypeScript errors with MCP SDK

Ensure you have the latest types:

```bash
npm install -D @types/node
```
