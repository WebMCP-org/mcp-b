Welcome! If you've read about [MCP-B's architecture](/blog/mcp-b-introduction), you're probably eager to start building. This guide will walk you through setting up your first MCP-B powered website.

## What You'll Build

By the end of this tutorial, you'll have:

- A web page that exposes MCP tools through `window.mcp`
- A working todo management system accessible via MCP
- A chat interface that can interact with your tools
- Understanding of how to add your own custom tools

## Prerequisites

Before starting, make sure you have:

- **Node.js 18+** installed on your machine
- **pnpm** or **npm** package manager
- Basic familiarity with **React** and **TypeScript**
- A text editor (VS Code recommended)

## Step 1: Project Setup

First, let's create a new project. MCP-B works with any modern web framework, but we'll use **TanStack Start** for this example:

<CodeBlock language="bash" code={`# Create a new project
pnpm create @tanstack/start

# Navigate to your project
cd my-mcp-app

# Install MCP-B dependencies
pnpm add @modelcontextprotocol/sdk`} />

## Step 2: Define Your MCP Tools

Create a file for your MCP server definition. This is where you'll define what tools your website exposes:

<CodeBlock language="typescript" code={`// src/services/mcp.ts
import { Server } from '@modelcontextprotocol/sdk/server/index.js'
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'

// Create the MCP server
export const mcpServer = new Server(
  {
    name: 'my-app-mcp',
    version: '1.0.0',
  },
  {
    capabilities: {
      tools: {},
    },
  }
)

// Define your first tool
mcpServer.setRequestHandler('tools/list', async () => {
  return {
    tools: [
      {
        name: 'get_greeting',
        description: 'Get a friendly greeting',
        inputSchema: {
          type: 'object',
          properties: {
            name: {
              type: 'string',
              description: 'Name to greet',
            },
          },
          required: ['name'],
        },
      },
    ],
  }
})

// Implement the tool
mcpServer.setRequestHandler('tools/call', async (request) => {
  if (request.params.name === 'get_greeting') {
    const name = request.params.arguments?.name
    return {
      content: [
        {
          type: 'text',
          text: \`Hello, \${name}! Welcome to MCP-B.\`,
        },
      ],
    }
  }
  throw new Error(\`Unknown tool: \${request.params.name}\`)
})`} />

## Step 3: Expose MCP to the Browser

Now let's make your tools available through `window.mcp`:

<CodeBlock language="typescript" code={`// src/main.tsx or your entry point
import { mcpServer } from './services/mcp'

// Expose the MCP server on window
declare global {
  interface Window {
    mcp: typeof mcpServer
  }
}

window.mcp = mcpServer

// Continue with your normal React app setup
// ...`} />

## Step 4: Add a Chat Interface

MCP-B really shines when you add a chat interface. You can use **@assistant-ui/react** for a quick setup:

<CodeBlock language="bash" code={`pnpm add @assistant-ui/react`} />

Then create a simple chat component:

<CodeBlock language="typescript" code={`// src/components/Chat.tsx
import { Thread } from '@assistant-ui/react'

export function Chat() {
  return (
    <div className="fixed bottom-4 right-4 w-96 h-[600px]">
      <Thread />
    </div>
  )
}`} />

## Step 5: Connect Everything Together

In your main app component, tie it all together:

<CodeBlock language="typescript" code={`// src/App.tsx
import { Chat } from './components/Chat'
import { AssistantRuntimeProvider } from '@assistant-ui/react'
import { useAssistantMCP } from './hooks/useAssistantMCP'

export function App() {
  const runtime = useAssistantMCP()

  return (
    <AssistantRuntimeProvider runtime={runtime}>
      <div className="min-h-screen">
        <header>
          <h1>My MCP-B App</h1>
        </header>

        <main>
          {/* Your app content */}
        </main>

        <Chat />
      </div>
    </AssistantRuntimeProvider>
  )
}`} />

## Step 6: Test Your Setup

Start your development server:

<CodeBlock language="bash" code={`pnpm dev`} />

Open your browser console and verify MCP is available:

<CodeBlock language="javascript" code={`// In browser console
console.log(window.mcp)
// Should show your MCP server object

// Try calling your tool
window.mcp.request({
  method: 'tools/call',
  params: {
    name: 'get_greeting',
    arguments: { name: 'World' }
  }
})`} />

## Common Patterns

### Adding More Tools

As your app grows, you'll want to add more tools. Here's a pattern for organizing them:

<CodeBlock language="typescript" code={`// src/services/mcp/tools/todos.ts
export const todoTools = [
  {
    name: 'list_todos',
    description: 'List all todos',
    inputSchema: {
      type: 'object',
      properties: {},
    },
  },
  {
    name: 'create_todo',
    description: 'Create a new todo',
    inputSchema: {
      type: 'object',
      properties: {
        text: { type: 'string' },
      },
      required: ['text'],
    },
  },
]

// Then import and use in your main MCP setup`} />

### Error Handling

Always wrap your tool implementations with proper error handling:

<CodeBlock language="typescript" code={`mcpServer.setRequestHandler('tools/call', async (request) => {
  try {
    // Your tool logic here
  } catch (error) {
    return {
      content: [
        {
          type: 'text',
          text: \`Error: \${error instanceof Error ? error.message : 'Unknown error'}\`,
        },
      ],
      isError: true,
    }
  }
})`} />

## Troubleshooting

### "window.mcp is undefined"

Make sure you're exposing the MCP server before your React app tries to use it. The exposure should happen in your main entry file.

### Tools not showing in chat

Verify that:
1. Your `tools/list` handler is returning the correct format
2. The tool names match between `list` and `call` handlers
3. Check the browser console for any errors

### TypeScript errors with MCP SDK

Ensure you have the latest types:

<CodeBlock language="bash" code={`pnpm add -D @types/node`} />
